# 📱 Doe Fácil – UX/UI Case Study

**UX/UI Case Study – Aplicativo mobile de doações com geolocalização**

Aplicativo mobile desenvolvido no Figma com foco em experiência do usuário, geolocalização e impacto social.

---

## 🌎 Visão Geral

O **Doe Fácil** é um projeto de UX/UI Design criado com o objetivo de facilitar a conexão entre doadores e instituições de caridade.

A proposta é tornar o processo de doação mais simples, rápido e confiável, utilizando geolocalização e uma interface intuitiva.

---

## 🎯 Problema

Muitas pessoas desejam doar, mas enfrentam dificuldades como:

- Falta de informação sobre instituições confiáveis  
- Dificuldade em encontrar locais próximos  
- Processos burocráticos ou pouco intuitivos  
- Ausência de acompanhamento das doações realizadas  

Essas barreiras reduzem o impacto social das doações.

---

## 💡 Solução

O **Doe Fácil** foi idealizado para:

- Conectar doadores a instituições próximas via geolocalização  
- Simplificar o processo de doação em poucos passos  
- Oferecer transparência através de relatórios e histórico  
- Proporcionar uma experiência acessível e intuitiva  

---

## 👥 Público-Alvo

### 🧍 Doadores
Pessoas que desejam doar itens em bom estado de forma prática e segura.

### 🏢 Instituições
ONGs e entidades que recebem e gerenciam doações.

### ⚙️ Administradores
Responsáveis pela gestão do sistema e relatórios.

---

## 🔁 Fluxo do Usuário

### Doador
Login → Buscar instituição → Visualizar detalhes → Doar item → Acompanhar histórico

### Instituição
Login → Gerenciar doações → Visualizar relatórios

---

## 🖥️ Principais Telas Desenvolvidas

- Tela inicial  
- Login com autenticação Google  
- Cadastro de usuário  
- Recuperação de senha  
- Listagem de instituições com busca e filtros  
- Mapa com geolocalização  
- Detalhamento da instituição  
- Histórico de doações  
- Relatório anual de impacto  

---

## 🎨 Decisões de Design

- Paleta azul para transmitir confiança e segurança  
- Interface limpa para reduzir distrações  
- Botões de ação destacados  
- Navegação inferior para facilitar acesso  
- Hierarquia visual clara e objetiva  

---

## 🔐 Requisitos de Segurança Pensados

- Criptografia de senhas  
- Autenticação segura (OAuth 2.0)  
- Controle de permissões por perfil  
- Proteção contra ataques comuns (SQL Injection, XSS, CSRF)

---

## 🧠 Arquitetura Planejada

- **Frontend:** React Native  
- **Backend:** Node.js ou Python  
- **Banco de dados:** PostgreSQL ou Firebase  
- **Infraestrutura:** AWS ou GCP  
- **Integração:** API de geolocalização (Google Maps)

---

## 📈 Metodologia

Projeto estruturado com metodologia ágil (Scrum), com:

- Sprints quinzenais  
- Entregas contínuas  
- Testes com usuários  
- MVP estimado em 3 meses
